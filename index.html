DataFlow Visualization

I want to do performant, interactive data visualization on the web.

I want to take as large dataset as possible, and create an interactive program that displays this data.

Interactive means more than just zoom / pan / select. I would like to allow for dynamic change of all kinds of parameters.

Performant means a million datapoints at 60fps on a 2015 MacBook Pro.

1 Investigate Technologies

I picked a very simple, yet not 100% trivial problem - display a 4D dataset.

My goal is to display as many points as I can, interactively.

3 The UI

Let's start with the flashiest part - a dataflow, visual programming, node editor inspired by the looks of Unreal Blueprint editor.

This is an important first step when selling the idea to people. So let's do the node editor justice.

For now I am shamelessly stealing and simplifying Виталий Рыбников's work. This is just to get off the ground quickly, not a long-term plan.

Once I have a nice visual editor, the whole idea of this setup would become more apparent than if I started with amazing backend, but did not allow people to experiment easily.

  • Node editor

4 The Interpreter

Once I can define node graph, I can start executing it.

Implement a simple interpreter that can take the nodes and connections between them and make the data flow, displaying the result.
